Ansible Infoblox NIOS 8.5 Lab

This lab deploys an Infbolox vNIOS 8.5 (NIOS) instance along with Ansible Tower (Tower) and VSCode. This can be used to demonstrate IP Administration (IPAM) or DNS automation with NIOS and Ansible. It can also be used as a sandbox for testing automation workflows with NIOS.

Working with the lab:

  • When writing playbooks, use the infoblox.nios_module collection
  • A new custom credential type called NIOS Custom Credentials is created in Tower
  • A new credential called nios creds with per-lab connection variables is predefined
  • A new inventory called nios will be populated with extra_vars specific to each lab
  • Use /var/lib/awx/venv/nios for ANSIBLE ENVIRONMENT in Tower or jobs will fail
  • Pass ansible_python_interpreter: /usr/bin/python3 to your playbook
  • A 10.0.0.0/8 network container is automatically added to NIOS
